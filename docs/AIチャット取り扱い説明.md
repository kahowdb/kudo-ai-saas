# Kudo AI チャット 取り扱い説明

WordPress サイトに設置する AI チャットの使い方・設定方法・システム構成です。

---

## 1. サイトマップ（システム構成）

### 1.1 全体の流れ

```
┌─────────────────┐     POST /train        ┌──────────────────┐
│  WordPress      │  ──────────────────►   │  Kudo AI SaaS    │
│  (プラグイン)    │  記事データ + X-Site-Id  │  (API サーバー)   │
│                 │                        │                  │
│  ・保存時に同期   │     GET /ask           │  ・チャンク分割    │
│  ・削除時に削除   │  ◄──────────────────   │  ・Embedding     │
│  ・一括同期      │  質問 + X-Site-Id       │  ・ベクトル検索    │
│                 │     回答 + 参照元        │  ・GPT 回答生成   │
└────────┬────────┘                        └────────┬─────────┘
         │                                           │
         │  REST /chat-config                         │  PostgreSQL
         │  (クイック質問・siteId)                     │  + pgvector
         ▼                                           ▼
┌─────────────────┐                        ┌──────────────────┐
│  フロント       │   X-Site-Id 付きで      │  ドキュメント     │
│  (チャットUI)   │   /ask を呼び出し       │  ・チャンク       │
│  ・クイック質問  │                        │  (サイト別に分離)  │
│  ・入力・送信   │                        └──────────────────┘
└─────────────────┘
```

### 1.2 構成要素一覧

| 要素 | 役割 |
|------|------|
| **WordPress プラグイン（Kudo AI Sync）** | 記事を API に送信・削除、設定画面、チャット UI の読み込み |
| **Kudo AI SaaS API** | 記事の保存・検索・質問への回答（RAG） |
| **PostgreSQL + pgvector** | 記事・チャンク・ベクトルの永続化（サイト別に `site_id` で分離） |
| **OpenAI API** | Embedding（text-embedding-3-small）とチャット（gpt-4o-mini） |

### 1.3 API エンドポイント一覧（Kudo AI SaaS）

| メソッド | パス | 説明 | 必須ヘッダー |
|----------|------|------|----------------|
| GET | `/health` | 死活確認 | なし |
| GET | `/documents` | 登録ドキュメント一覧 | X-Site-Id（未指定時は default） |
| GET | `/documents/:id` | 1件取得 | X-Site-Id |
| GET | `/chunks` | チャンク一覧（?documentId= で絞り込み可） | X-Site-Id |
| POST | `/train` | 記事の登録・更新（チャンク化・Embedding） | X-Site-Id |
| POST | `/ask` | 質問に回答（ベクトル検索 + GPT） | X-Site-Id |
| POST | `/delete` | 記事の削除（body: `{ "id": 投稿ID }`） | X-Site-Id |

### 1.4 WordPress プラグインの主な機能

| 機能 | 説明 |
|------|------|
| 設定画面 | 接続先URL、同期ON/OFF、対象投稿タイプ、固定チャット、**クイック質問・回答・リンク** |
| 記事保存時の同期 | 公開済みの投稿・固定ページを保存すると自動で `/train` に送信 |
| 記事削除時の削除 | ゴミ箱送信などで削除すると `/delete` に送信 |
| 過去記事の一括同期 | 設定画面から「過去記事を全件同期」で既存記事をまとめて送信（最大500件/回） |
| 手動同期（1件） | 投稿IDを指定してその記事だけ即時送信 |
| 右下固定チャット | 有効にすると全ページにチャットウィジェットを表示 |
| ショートコード | `[kudo_ai_chat]` で任意のページにチャットを埋め込み |
| クイック質問 | ボタンで表示する質問文と、任意の「回答＋リンクURL」を設定可能 |

## 2. 取り扱い説明（管理者向け）

### 2.1 初回設定

1. WordPress 管理画面で **「設定 → Kudo AI Sync」** を開く。
2. 次を入力・選択して **「設定を保存」** をクリック。
   - **接続先URL（記事同期用）**  
     記事を送る先の Kudo AI SaaS の URL（例: `https://your-ai-api.example.com`）。  
     ※ あなたの WordPress の URL ではありません。
   - **接続先URL（チャット用）**  
     サイト訪問者のブラウザからアクセスする API の URL。通常は上と同じでよい。
   - **同期対象の投稿タイプ**  
     「投稿」「固定ページ」など、AI に学習させたいものをチェック。
   - **同期をON** にチェック。
   - チャットを使う場合は **「右下固定のAIチャットを表示する」** にチェック。
3. 既存記事を学習させたい場合は **「過去記事を全件同期する」** を実行。

※ 複数サイトで同じ API を使う場合、**サイトごとにデータが分離**されます（各サイトの URL が `X-Site-Id` として使われます）。

### 2.2 クイック質問・回答の設定

「設定 → Kudo AI Sync」の **「クイック質問・回答」** で、チャットに表示するボタンと、任意の「回答＋リンク」を設定できます。

| 項目 | 説明 |
|------|------|
| **質問（ボタンに表示）** | チャットに表示する文言（例: 制作費の目安は？） |
| **回答** | その質問用の定型文。空なら AI が記事から回答します。 |
| **リンクURL** | 誘導したいページの URL（絶対URLや `/price` など相対でも可）。 |

- **回答とリンクの両方またはどちらかを入力した場合**  
  その質問をクリックすると、API を呼ばずに設定した回答とリンクだけを表示します。
- **両方空の場合**  
  従来どおり AI が記事を検索して回答します。

### 2.3 同期のタイミング

- **自動**  
  対象の投稿タイプを「保存」すると、その記事が API に送信されます。削除すると API からも削除されます。
- **一括**  
  「過去記事を全件同期する」で、公開済みの対象記事をまとめて送信（1回あたり最大500件。多い場合は再度実行）。
- **手動（1件）**  
  投稿IDを入力して「今すぐ同期する」で、その1件だけ送信。

### 2.4 よくある注意点

- **接続先URL** は「AI 用のサーバー」の URL です。WordPress のサイト URL ではありません。
- Docker 内の WordPress から同期する場合、接続先は `http://host.docker.internal:8787` のような形になることがあります。
- チャット用の URL は、**訪問者のブラウザから実際にアクセスできる URL** を設定してください。
- 設定を変えてもチャットのクイック質問が変わらない場合は、**強制再読み込み（Ctrl+Shift+R / Cmd+Shift+R）** を試してください。

## 3. 取り扱い説明（サイト訪問者・利用者向け）

- 右下のチャットアイコン（💬）をクリックするか、チャットが埋め込まれたページではそのまま入力欄が表示されます。
- **クイック質問**のボタンを押すと、その質問が送信され、AI がサイトの記事をもとに回答します（設定によっては定型文とリンクだけが表示されます）。
- 自由に文章を入力して送信することもできます。サイト内の記事を参照して回答します。
- 回答の下に **参照元** が表示される場合は、そこから該当記事に飛べます。

### Kudo AI SaaS（API）

```
kudo-ai-saas/
├── src/
│   ├── server.ts   # ルート・/ask・/train・/delete など
│   ├── db.ts       # PostgreSQL + pgvector
│   └── types.ts    # 型定義
├── docs/
│   └── AIチャット取り扱い説明.md  # 本ドキュメント
├── package.json
└── .env            # OPENAI_API_KEY, DATABASE_URL など
```

### WordPress プラグイン（Kudo AI Sync）

```
kudo-ai-sync/
├── kudo-ai-sync.php   # メイン（設定・同期・REST・チャット出力）
├── js/
│   └── chat.js        # チャットUI・/ask 呼び出し・クイック質問
└── css/
    └── chat.css       # チャットのスタイル
```
